Connector sync & load handling — thousands of users, isolated

My integrations · Multi-tenancy

My connectors push/pull strings on a schedule via the REST API (API key → JWT). Under load the platform stays calm by design: throttle at the edge, serve reads from cache & Elasticsearch, and push heavy work to the job queue — so a spike just lengthens a queue instead of stalling the web tier. Each tenant's data lives in its own database, so one company's load never touches another's.

Two project models — manual Standard jobs vs automated Codyt stream

Projects · Workflow

Standard suits one-off batches: the PM creates each job by hand and advances it. Codyt suits living content (websites, apps): you define the workflow once and the system generates tasks per segment and per language whenever new content arrives — no repeat setup.

Versioned re-import — only changed segments become work

Word count · Segments

When living content updates, the connector re-imports a new version of the same document. Word count compares it against the previous version: unchanged sentences inherit the old translation automatically; only new or edited sentences are flagged and turned into tasks. Clients pay for changed text only.
