Wordbee Connectors · Overview

What does each source contribute to Wordbee?

Wordbee Translator (WB) is the central translation management platform. The three connectors — Google Sheets, Unity and Unreal — don't replace one another; they all do the same job: push source content to WB for translation and pull translations back in each platform's native format. The difference lies in where the content lives.

The shared flow across all three

auth Authenticate

Enter API URL, Account ID, API Key → obtain an AuthToken via the /api/auth/token endpoint.

link Link the document

Select or enter a Document ID in WB — this is the document the source will sync with.

push Push source content

Send source strings (and any existing translations) to WB. Optionally push only what changed.

translate Translate in WB

Translators, workflows and QA handle the segments right on the Wordbee platform.

pull Pull translations back

Retrieve translations by the selected language and segment-status filter.

02

What each source uniquely contributes

GS

Google Sheets V2 — FlexConnector

Google Workspace Add-on (Apps Script) that runs right inside the spreadsheet

Contributes spreadsheet-based content to WB — UI strings, dialogue, and content tables that the team edits directly in Google Sheets. The add-on adds Synchronization and Configure menus to Sheets, turning each sheet into a two-way translation document.

Push to WB

  • Export the active sheet as an .xlsx file
  • Upload via the media API (multipart), then call pushChangesToFlex
  • WB uses the configured parser/format to split segments

Pull back to Sheets

  • pullAndConvertSegments using the segment filter
  • Poll the operation status, download the file, convert segments → rows
  • Merge by key (mergeSheetRowsByKey) or replace the entire sheet content

Distinctive traits

  • No installation required — runs entirely in the browser using the spreadsheets + script.external_request scopes.
  • Status filter: All segments / exclude errors / approved only.
U

Unity Connector

Editor extension integrated with the Unity Localization system

Contributes Unity game localization strings to WB, stored in a StringTableCollection. Opened through an Editor window with Connect, Document Info, Config, Key Viewer and Workflow Status tabs — and it also serves translated strings right at runtime.

Push to WB

  • TblCollectionUtils.GetChangesSince retrieves changed records
  • Filters by the locales common to both Unity and WB
  • Pushes source strings + translations from the selected collection

Pull back to Unity

  • API.PullDocument updates the String Table directly
  • Filters: All / Error-free (IfAnyProblem) / Approved only (IfNotAllOk)
  • Periodic auto-sync on a preconfigured interval

Distinctive traits

  • Automatically detects Missing Languages — locales present in Unity but not yet in WB.
  • Runtime API WordbeeLocal.GetStringData(recordID) returns strings for the active target language.
UE

Unreal Connector — WordbeeEditor

C++ plugin for the Unreal Editor (UE 5.6)

Contributes the Localization Target / Namespace of an Unreal project to WB. The plugin hooks into the Tools ▸ Custom Tools ▸ Wordbee Link menu with three items: Configure, Key Viewer and Workflow Status.

Push to WB

  • DocumentService::PushDocument pushes source strings / translations
  • Push only changed data option
  • Matches by Target Name + Namespace (default Game)

Pull back to Unreal

  • DocumentService::PullDocument for the selected language
  • Filters: All / exclude error segments / approved only
  • Auto-sync every 1–30 minutes via CronJobHandler

Distinctive traits

  • Supports composed keys of the form Namespace:::Key with a customizable separator.
  • The auth token is saved to the plugin's .ini file after Test Credentials.
