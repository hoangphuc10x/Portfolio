Raydium swap — route by pool type (AMM · CLMM · CPMM)

RaydiumService.performSwap · SDK v2

POST /api/raydium/swap finds the best pool for the mint pair, reads decimals to scale the raw amount, computes the slippage-bounded minimum output, then builds a V0 transaction and submits it. The pool kind decides which SDK namespace handles compute + swap. AMM (raydium.liquidity.*) is implemented today; CLMM and CPMM are the planned routes on the radium-swap branch.

Scheduled auto-trading — per-schedule buy/sell cron

TxScheduleService · SchedulerRegistry

Creating a schedule registers two dynamic cron jobs (tx-schedule:{id}:buy / :sell) at */{periodSeconds} * * * * *. Each tick picks an eligible wallet, checks for pool migration, and fires a swap. On boot, on-app-start re-arms every enabled schedule (production only).

Single NestJS backend (no separate frontend in repo). Clients reach it over REST /api + Socket.IO; it persists to MongoDB, caches / queues / locks in Redis (BullMQ · Redlock · Bottleneck), submits Solana transactions through Shyft / Helius / Yellowstone gRPC and the Astralane MEV relay, and offloads auth, email, IPFS & scraping to external services. Deployed via GitHub Actions → SSH → PM2.
